# PMM-RTE Reproducibility Supplement

Open-code reproducibility bundle for:

> **Polynomial-Maximization Ridge-Type Estimation for Multicollinear Regression
> under Non-Gaussian Errors**
> Serhii Zabolotnii et al.

It reproduces the manuscript's tables and figures for **PMM-RTE** — the composition
of a Kunchenko polynomial-maximization estimator (PMM2/PMM3) with the ridge-type
estimator (RTE) of Akay et al. (REVSTAT v24-n1-3) for multicollinear linear
regression. The headline claim concerns **coefficient stability (β-space variance)**;
prediction RMSE is only a guardrail.

```
beta_PMM_RTE = L_RTE(k, g; X) · beta_PMM      (RTE is a linear operator on any base estimate)
Cov(beta_PMM_RTE) ≈ g_S · Cov(beta_RTE)        (covariance transfer; g_S = g2 or g3)
```

The multi-megabyte per-replication CSV/RDS dumps are **not** shipped (they are
regenerated by the scripts); only the compact summaries the manuscript cites are stored.

## Requirements

- **R ≥ 4.5** with the workspace core package **`EstemPMM`** installed (provides
  `lm_pmm2()` / `lm_pmm3()`). All other dependencies are base/recommended packages
  (`stats`, `utils`, `graphics`, `grDevices`, `parallel`).
- Run **from the repository root** — the scripts resolve `R/config.R` and `results/`
  by relative path.

## Quick reproducibility check

Regenerate tables T1–T5 and the verification gates **from the stored summaries**
(fast; no heavy Monte Carlo):

```bash
Rscript --vanilla scripts/04_make_tables.R
Rscript --vanilla scripts/05_verify_results.R
```

`05_verify_results.R` asserts the SPEC acceptance gates against the stored summary
CSVs and exits non-zero if a hard gate fails.

## Full rerun

```bash
# screening grid (fast go/no-go); set REALDATA=1 to include the real-data block
PHASE=screening Rscript --vanilla run_all.R

# publication grid (heavy)
PHASE=final REALDATA=1 Rscript --vanilla run_all.R
```

Useful environment switches (defaults in `R/config.R`):

| var | meaning |
|-----|---------|
| `PHASE` | `screening` (default) or `final` — reps / test_n / scenario grid |
| `G_MODE` | `deployable` (default, `g ≥ 0` headline), `full_sens`, or `coupled` |
| `SYM` | `1` (default) include the symmetric / PMM3 block |
| `REALDATA` | `1` include the slower real-data repeated-split block (default `0`) |
| `REPS`, `TEST_N`, `B_BOOT` | numeric overrides for smoke runs |

Verification is by reproduction: re-run a script and diff the regenerated
`results/*_report.md` against the stored copy. **Never hand-edit files in `results/`** —
regenerate them.

## Method grammar (base × transform × tuning)

- **base** ∈ `OLS`, `PMM2`, `PMM3`
- **transform** ∈ `Ridge` (`g = 1`), `RTE` (tune `k` and `g`); the intercept is never penalized
- **tuning** ∈ `CV` (min inner-validation RMSE), `Stable` (smallest slope L2-norm within
  the RMSE guard band), `Oracle` (min β-MSE — **synthetic only, non-deployable**, reported
  as an upper bound)

Dispatch by residual cumulants (`gamma3, gamma4, gamma6` → `g2, g3`): PMM2 when
`|gamma3| > 1.0`, PMM3 when `|gamma3| < 0.3 & gamma4 < -0.7`, else OLS/RTE — gated by a
sample-size rule (`n < 50` ⇒ illustrative only). Portland/Hald cement (`n = 13`) is a
deliberate grey-zone illustration, not a PMM proof case.

## Scope and caveats

- **Main proof** = synthetic Monte Carlo with known `beta` (β-MSE). **Real data** is
  supporting evidence: it shows coefficient stabilization with a scale-dependent RMSE
  tradeoff, not a universal prediction win.
- Under **Normal** errors PMM makes no strong claim, and PMM-RTE must not lose to RTE by
  more than ~10 % in β-MSE (Gaussian-fallback gate).
- Oracle tuning is an upper bound, never a deployable claim.
